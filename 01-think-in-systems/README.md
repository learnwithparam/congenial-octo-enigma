# 01 -- Think in Systems

Problem Decomposition and System Design for Builders.

## What You'll Learn

How to break down a product idea into user stories, features, data models,
API contracts, component trees, and a technical spec -- before writing any code.

## Prerequisites

- No coding prerequisites
- Curiosity about how software systems fit together

## Lessons

| # | Lesson | Topic |
|---|--------|-------|
| 01 | [what-were-building](./lessons/01-what-were-building/) | Introduce the project and define the vision |
| 02 | [user-stories](./lessons/02-user-stories/) | Write user stories that drive development |
| 03 | [feature-breakdown](./lessons/03-feature-breakdown/) | Decompose features into manageable pieces |
| 04 | [data-modeling](./lessons/04-data-modeling/) | Design the data model for your application |
| 05 | [api-contracts](./lessons/05-api-contracts/) | Define clear API contracts between layers |
| 06 | [component-tree](./lessons/06-component-tree/) | Map out the frontend component hierarchy |
| 07 | [architecture-decisions](./lessons/07-architecture-decisions/) | Make and document architecture decisions |
| 08 | [technical-spec](./lessons/08-technical-spec/) | Pull everything together into a technical spec |
